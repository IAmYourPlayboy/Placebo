# 3D World Visual Polish ‚Äì Clean Dark Style

**Date:** 2026-03-16
**Status:** Approved
**Scope:** Make Tokyo scene beautiful with quality presets (low/medium/high)

## Context

Placebo's 3D world currently has placeholder visuals: dark circle ground, 50 mock buildings, flat lighting, solid-color sky. The goal is to polish the scene into a clean, minimalist "digital frame" for live camera video ‚Äì prioritizing real data over decoration.

### Design Principles

- **Real data only** ‚Äì no fake buildings. Empty world is better than fake.
- **Clean Dark aesthetic** ‚Äì the world is a frame for video, not the star
- **Three quality presets** ‚Äì low (weak PCs), medium (default), high (powerful GPUs)
- **Layered architecture** ‚Äì each visual system is an independent module
- **Future-proof** ‚Äì style system designed for eventual marketplace of visual themes

### User Decisions

- Visual style: **Clean Dark / Minimalism** (option B)
- Ground: **Real roads from OSM** via our pipeline + API (option B, server-side)
- Sky elements: gradient + stars + sun/moon (low/medium), + clouds + atmospheric (high)
- Lighting: basic (low), + SSAO/bloom (medium), + volumetric (high)
- No fake/procedural buildings ever
- Post-effects minimal: only for medium/high presets

## 1. Quality Preset System

### Type Definition

**Replaces** existing `QualitySettings` in `src/types/world3d.ts`. The old interface (`lodMultiplier`, `loadRadius`, `wireframeEnabled`, `postProcessing`, `maxVideoTextures`) is removed entirely. All consumers (WorldScene, BuildingsLayer) are updated to use the new `QualityConfig`.

```typescript
type QualityPreset = 'low' | 'medium' | 'high';

type SkyMode = 'gradient' | 'gradient-stars-sun' | 'atmospheric';
type GroundDetail = 'roads-basic' | 'roads-detailed' | 'roads-ssr';
type LightingMode = 'basic' | 'ssao-bloom' | 'full';
type PostMode = 'none' | 'tonemap-vignette' | 'full';

interface QualityConfig {
  preset: QualityPreset;
  sky: { mode: SkyMode };
  ground: { detail: GroundDetail; gridEnabled: boolean };
  lighting: { mode: LightingMode; shadowMapSize: number };
  post: { mode: PostMode };
  maxVideoTextures: number;
  fog: { near: number; far: number; color: string };  // varies by preset + time
}
```

### Preset Values

| Parameter | Low | Medium | High |
|---|---|---|---|
| `sky.mode` | `gradient` | `gradient-stars-sun` | `atmospheric` |
| `ground.detail` | `roads-basic` | `roads-detailed` | `roads-ssr` |
| `ground.gridEnabled` | `false` | `true` | `true` |
| `lighting.mode` | `basic` | `ssao-bloom` | `full` |
| `lighting.shadowMapSize` | 1024 | 2048 | 4096 |
| `post.mode` | `none` | `tonemap-vignette` | `full` |
| `maxVideoTextures` | 1 | 2 | 4 |

### Storage

Preset saved in Tauri SQLite (`settings` table), read via IPC `get_setting("quality_preset")`. Default: `medium`.

### React Context

`QualityContext` + `useQuality()` hook. Components subscribe to relevant fields. Preset change rebuilds config, components react.

## 2. Ground System

### Data Pipeline (offline, once per city)

```

```

Output per city:
```
pipeline/output/tokyo/
  tileset.json       ‚Äî 3D Tiles buildings (existing)
  roads.geojson      ‚Äî road network (NEW)
```

### API Endpoint

```
GET /api/world/roads?lat=35.6595&lng=139.7005&radius=1000
```

Axum handler serves **static GeoJSON files** from `pipeline/output/{city}/roads.geojson`. No PostGIS queries at runtime ‚Äì files are pre-generated by the pipeline.

Response schema:
```typescript
// GeoJSON FeatureCollection
{
  type: "FeatureCollection",
  features: [{
    type: "Feature",
    geometry: { type: "LineString", coordinates: [[lng, lat], ...] },
    properties: {
      highway: "primary" | "secondary" | "residential" | "footway" | ...,
      name: string | null,
      lanes: number | null,
      width: number | null  // meters, from OSM tag or null
    }
  }, ...]
}
```

### Road Rendering

Convert lat/lng to local XZ via `geoToLocal()`. Each road LineString is extruded into a flat ribbon mesh using custom `BufferGeometry`: for each pair of consecutive points, compute the perpendicular direction, offset by half-width on each side to create a quad (2 triangles). At corners, use miter join (average of adjacent perpendiculars). No external library needed ‚Äì simple geometry for flat roads on XZ plane.

Width by road type:

| Road Type | Width (m) | Color (Clean Dark) |
|---|---|---|
| `primary` / `trunk` | 12‚Äì15 | `rgba(255,255,255, 0.06)` |
| `secondary` | 8‚Äì10 | `rgba(255,255,255, 0.045)` |
| `residential` | 6 | `rgba(255,255,255, 0.03)` |
| `footway` / `path` | 2‚Äì3 | `rgba(255,255,255, 0.02)` |

Intersections: subtle glow in camera accent color (`#E8345A`, opacity 0.05).

### Ground Plane

- Color: `#0a0e14` (slightly blue, deeper than current)
- Grid overlay: 50m cells, opacity 0.02 (medium/high only)
- Fade to edges (opacity drops toward 2000m radius)

### By Preset

**Low:** `SkyGradient` only ‚Äì improved SkyDome with vertical gradient (dark zenith ‚Üí lighter horizon). Smooth phase transitions via lerp.

**Medium:** + `StarField` (Points geometry, ~500 particles, opacity tied to time of day) + `CelestialBodies` (sun Sprite with radial gradient glow at directional light position, moon 20:00‚Äì6:00).

**High:** + `CloudLayer` (5-8 PlaneGeometry at ~800m with procedural alpha texture, slow movement at 0.5 m/s) + optionally `AtmosphericSky` (drei Sky with Preetham model, muted parameters for Clean Dark).

### Shared Time Hook

Extract time-of-day logic from current `Environment.tsx` into:

```typescript
// src/hooks/useTimeOfDay.ts
useTimeOfDay(timezone: string) ‚Üí {
  hour: number;          // 0-24 continuous
  sunPosition: Vector3;  // position on sky arc (see formula below)
  phase: 'night' | 'dawn' | 'morning' | 'day' | 'dusk' | 'twilight';
  t: number;             // 0-1 progress within current phase (for lerp)
}
```

**Sun position formula** (same approach as current `WorldLights`, formalized):
- Sun travels an arc from east (+X) at 6:00 to overhead (+Y) at 12:00 to west (-X) at 18:00
- `angle = ((hour - 6) / 12) * PI` (0 at 6AM, PI at 6PM)
- `sunPosition = Vector3(cos(angle) * R, sin(angle) * R, 0)` where R = 1000
- Below horizon (hour < 6 or > 18): sun stays below y=0, directional light intensity drops to minimum
- Moon uses opposite arc: `moonAngle = ((hour - 18) / 12) * PI`

All sky and lighting components use this hook.

## 4. Lighting System

### Files

```
src/components/world3d/lighting/
  LightingSystem.tsx   ‚Äî orchestrator
  BasicLights.tsx      ‚Äî ambient + directional (all presets)
  NightLights.tsx      ‚Äî street lights along roads (all presets)

src/components/world3d/post/
  PostStack.tsx        ‚Äî conditional post-processing effects
```

### BasicLights (all presets)

Improved WorldLights:
- Colors and intensities from `useTimeOfDay()` ‚Äì continuous, not stepped
- Ambient: warm tint at dawn/dusk, cool at night, neutral daytime
- Directional: real sun arc with elevation
- Shadows: `shadowMapSize` from preset config

### NightLights (all presets)

Point lights placed along major roads. Appear when `hour > 18 || hour < 6`.

**Cross-system dependency:** `NightLights` receives road positions via a shared `RoadData` interface exposed by `useRoadNetwork()`. It does NOT import ground components ‚Äì only consumes the hook's data (array of road center-points with type). If roads haven't loaded yet, NightLights renders a hardcoded fallback grid of 4 lights at fixed positions.

- **Low**: 4-6 lights on primary roads (sample every ~200m along road)
- **Medium**: 10-15 lights (sample every ~100m)
- **High**: more lights + faint colored spots (signage imitation)

Warm color `#ffaa44`, small radius (30m), decay 2.

### PostStack (medium/high only)

**Medium:**
- SSAO: `radius: 0.1`, `intensity: 1.5`
- Bloom: `threshold: 0.9`, `intensity: 0.3`
- ToneMapping: ACES Filmic

**High:**
- All of Medium
- Vignette: `offset: 0.3`, `darkness: 0.6`
- Lens flare on sun (custom or simplified sparkle)
- Optional: god rays (volumetric scattering, separate toggle)

### Dependency

`@react-three/postprocessing` + `postprocessing` ‚Äì needed for medium/high. Low works without. Use dynamic `React.lazy()` import in `PostStack.tsx` to avoid bundling for low preset. While loading, render nothing (no visual flash ‚Äì effects are additive, their absence is not noticeable).
